**Weave.js** is an open-sourced and growing project, and we welcome contributions
from developers, designers, technical writers, and the broader community!

Your ideas, improvements, bug fixes, and new features help make Weave.js better,
stronger, and more useful for all the community.

This section explains how external collaborators can get involved, collaborate
with the project, and make an impact.

## Ways to Collaborate

| Area                     | How You Can Contribute                                                                                     |
| ------------------------ | ---------------------------------------------------------------------------------------------------------- |
| **Code Contributions**   | Fix bugs, improve performance, build new features, or extend plugin and store support.                     |
| **Documentation**        | Help us improve or expand the docs: quickstarts, tutorials, architecture deep-dives, or translations.      |
| **Testing and QA**       | Try new releases, report issues, suggest improvements for usability, performance, or developer experience. |
| **Ideas and Proposals**  | Suggest new tools, plugins, integrations, or use cases by opening discussions.                             |
| **Community Engagement** | Answer questions, write blog posts or tutorials about Weave.js, help other developers get started.         |

## How to start contributing

1. **Check the open issues**: browse the [GitHub Issues](https://github.com/InditexTech/weavejs/issues)
   to find open bugs, enhancement requests, or tasks labeled "candidate".
2. **Fork the repository**: fork the [Weave.js GitHub repository](https://github.com/InditexTech/weavejs)
   and create a new branch for your contribution.
3. **Follow the development guidelines**: please read the [CONTRIBUTING.md](https://github.com/InditexTech/weavejs/blob/main/CONTRIBUTING.md)
   file for coding standards, commit style, and pull request best practices.
4. **Submit a pull request**: open a [pull request](https://github.com/InditexTech/weavejs/pulls)
   (PR) with a clear description of what you changed or added. The core team will
   review it, suggest feedback if necessary, and merge when ready.
